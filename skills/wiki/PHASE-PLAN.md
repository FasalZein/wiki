# Phase: plan

Goal: turn a vague request into bounded work the vault can track.

1. Read the current project status and relevant ADRs.
2. Ask only blocking questions; prefer concrete options over open-ended prompts.
3. Capture constraints, risks, and success criteria.
4. Convert settled scope into PRD-ready language.
5. Do not create implementation slices until the PRD is clear enough to test.
