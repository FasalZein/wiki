# Admin: migration

Use this when moving knowledge from v1 into wiki-v2.

1. Identify the source vault/project and the target wiki-v2 project.
2. Inventory PRDs, slices, decisions, handovers, and research notes before writing.
3. Prefer CLI import or create/set verbs so templates and schemas stay valid.
4. Preserve original IDs in body text when IDs must change.
5. Run `wiki doctor` after migration and fix reported issues before closing the work.
